# KiraAI_auto_delete_session_plugin（ADS）

# 自动检测token重开对话插件 1.3.2

[![Ask DeepWiki](https://deepwiki.com/badge.svg)](https://deepwiki.com/znq19/KiraAI_auto_delete_session_plugin)

当会话新增 token 估算超过阈值时自动删除该会话并保留最近消息轮，释放token和不易超限，保持回复速度。

webui可设置阀值和估算比例、检查间隔、保留消息轮数。新增token即不含原本人设、工具注入等的花费。

注：

1.如感觉多次重开后token下降不明显，可降低保留消息轮数（例如降为5以下或直接为0）。因为有时候，你的工具调用（例如网络搜索）一轮就占用了非常多的token，保留最近轮数依然token庞大。

2.对于害怕破坏缓存命中的用户，应当优先将保留消息轮数设为0，或设置更高的token上限以不频繁触发重开。

## 重开前自动摘要

可在 webui「重开前自动摘要」区开启：重开时把将被清理的较早历史压缩成一段摘要，写入新会话首条记忆（`[前情摘要|系统注入]` 标记），让 bot 重开后仍能衔接上文。

- **off**：不摘要（原行为）
- **sync**：重开前同步生成摘要（阻塞本轮几秒，重开后第一轮即可看到摘要）
- **async**：先重开不阻塞，摘要后台生成后补写进新会话首部（重开后第一轮回复看不到摘要）

可选摘要模型（不选自动回退：快速模型 → 默认模型）、超时、输入上限与提示词。

**连续降级复用：** 若短时间（检查间隔的一半内）连续触发重开，自动读取上次已写入首条的摘要复用，避免重复调 LLM。

**「摘要详细日志」** 开关可输出触发条件、复用判断、完整摘要文本等调试信息。

## 连续超限动态降级

若短时间（检查间隔的一半内）连续触发重开，保留轮数自动减半（最少 1 轮），防止频繁超限后对话被截得过于零碎。

## 压缩重开命令

在「压缩重开命令」区配置（默认关闭）：

| 配置 | 说明 |
|------|------|
| 命令关键词 | list，整句匹配任一词即触发，默认 `/resum` |
| 权限控制 + 白名单 | 参考 reboot 插件：未开权限 / 白名单为空 → 所有人可用；开启后仅白名单用户 ID 可执行 |
| 成功/权限不足/错误文案 | 支持 `{keep}` `{summary}` `{error}` 占位符 |

发送命令后插件立即对当前会话执行「压缩重开」（先为被清理的较早历史生成摘要，再重开会话并保留最近轮数）并直接回复结果；与 reboot 类插件的全清空不同，压缩重开会保留摘要与最近对话。命令消息本身不会进入 LLM 上下文（与其他自定义命令一致）。摘要行为同样受上方「摘要模式」控制（off = 只重开不摘要）。
